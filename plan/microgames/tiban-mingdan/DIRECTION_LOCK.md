# Direction Lock: 替班名单

## One Sentence
玩家安排替班，让工分达标又不把关键工人压垮。

## Core Loop
1. 查看工人状态 → 拖到岗位 → 预览工分/疲劳 → 确认排班 → 结算崩溃/举报
2. 按 20 分钟节奏推进（4 班次）：基础排班 → 关系影响 → 病号和危险岗 → 工头指定名单

## Required State（定义与范围）

| State | 类型 | 范围 | 含义 |
|-------|------|------|------|
| quota | number | 0–100 | 当前班次需达成的工分目标百分比 |
| fatigue | per-worker | 0–10 | 工人疲劳值；≥8 有崩溃风险，=10 崩溃 |
| job_risk | per-job | 0–5 | 岗位危险等级；叠加疲劳产生事故概率 |
| resentment | per-pair | 0–10 | 两工人间怨恨值；≥8 触发举报事件 |
| shift | enum | pending / confirmed / resolved | 当前班次阶段 |

## Success
- 所有班次 quota 达标，无工人崩溃，无举报触发 → 良好结局
- 至少一个班次 quota 达标且无崩溃 → 可结算结局

## Failure
- 任意工人 fatigue 达到 10 → 该工人崩溃 → 若为关键工人则游戏结束
- 任意工人对 resentment 达到 10 → 举报事件 → 被系统淘汰
- 班次 quota 连续未达标 → 被工头淘汰

## Must Keep
- 核心机制：拖卡排班 + 疲劳扩散 + 怨恨网络
- 核心循环不变
- 20 分钟结构只作节奏，不扩成大项目

## Must Not Add
- 不做公司排班；核心是压迫劳动下的替班
- 不新增第二套主循环
- 不把小游戏扩成长期经营或开放世界

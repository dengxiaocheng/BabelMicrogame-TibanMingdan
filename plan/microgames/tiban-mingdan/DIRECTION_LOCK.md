# Direction Lock: 替班名单

## One Sentence
玩家安排替班，让工分达标又不把关键工人压垮。

## Core Loop
1. 执行核心循环：查看工人状态 -> 拖到岗位 -> 预览工分/疲劳 -> 确认排班 -> 结算崩溃/举报
2. 按 20 分钟节奏推进：基础排班 -> 关系影响 -> 病号和危险岗 -> 工头指定名单

## Must Keep
- 核心机制必须保持：拖卡排班 + 疲劳扩散 + 怨恨网络
- 核心循环必须保持：查看工人状态 -> 拖到岗位 -> 预览工分/疲劳 -> 确认排班 -> 结算崩溃/举报
- 20 分钟结构只作为节奏，不扩成大项目

## Must Not Add
- 不做公司排班；核心是压迫劳动下的替班
- 不新增第二套主循环
- 不把小游戏扩成长期经营或开放世界

## Required State
- resource
- pressure
- risk
- relation
- round

## Success
在限定时长内完成主循环，并稳定进入至少一个可结算结局

## Failure
关键状态崩溃，或在本轮主循环中被系统淘汰

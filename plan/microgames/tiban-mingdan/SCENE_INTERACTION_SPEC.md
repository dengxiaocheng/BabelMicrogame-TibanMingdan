# SCENE_INTERACTION_SPEC: 替班名单

## Scene Layout

```
┌─────────────────────────────────────────────────┐
│  班次信息栏 (shift, quota 目标, 剩余岗位)        │
├──────────────────┬──────────────────────────────┤
│                  │                              │
│  工人卡片区      │    岗位槽区                    │
│  (可拖拽)        │    (拖放目标)                  │
│                  │                              │
│  [工人A卡]       │    [岗位1槽]  job_risk: ★★   │
│  [工人B卡]       │    [岗位2槽]  job_risk: ★★★  │
│  [工人C卡]       │    [岗位3槽]  job_risk: ★    │
│  [工人D卡]       │    [岗位4槽]  job_risk: ★★★★ │
│  [工人E卡]       │                              │
│  [工人F卡]       │                              │
│                  │                              │
├──────────────────┴──────────────────────────────┤
│  预览面板 (quota预估 / fatigue增量 / 怨恨变化)   │
├─────────────────────────────────────────────────┤
│  [确认排班]  按钮                                │
└─────────────────────────────────────────────────┘
```

## Scene Objects

### 工人卡（Worker Card）
- 显示：名字、fatigue 值、特长图标
- 交互：可拖拽（drag source）
- 状态指示：fatigue ≥ 6 黄色边框，≥ 8 红色边框
- 拖拽中：半透明跟随光标，原位显示空位
- 已放入岗位：卡移到岗位槽内，卡片区显示灰色占位

### 岗位槽（Job Slot）
- 显示：岗位名、job_risk 等级（星星或危险图标）
- 交互：拖放目标（drop target）
- 空槽：虚线边框，显示"拖入工人"
- 已填入：显示工人卡缩略 + 岗位危险标记
- hover 预览：工人卡悬停时预览该工人放入的 fatigue 增量

### 预览面板（Preview Panel）
- quota 进度条：当前预估 / 目标值
- fatigue 增量列表：每个被排工人的 fatigue 变化
- resentment 变化列表：新增的怨恨关系
- 颜色规则：超标红，警告黄，正常绿

### 班次信息栏（Shift Info Bar）
- 当前班次编号（第 X 班 / 共 4 班）
- quota 目标百分比
- 剩余空岗位槽数

## Interaction Flow

1. 拖工人卡到岗位槽 → 卡移到槽内，预览面板更新
2. 从岗位槽拖回工人卡 → 撤销排班
3. 拖工人A到工人B所在岗位槽 → A 取代 B，B 回到卡片区
4. 所有岗位槽填满 → 确认按钮激活
5. 点击确认 → 结算动画 → state 更新 → 下一班次或结局

## Feedback Channels

| 通道 | 触发 | 表现 |
|------|------|------|
| 岗位收益预览 | 拖入/拖出岗位槽 | 预览面板 quota 数字实时更新 |
| 疲劳增量 | 拖入岗位槽 | 工人卡上 fatigue 数值闪动 + 颜色变化 |
| 怨恨连线 | 拖入替换上一班工人 | 两工人之间出现/加深红色连线 |
| quota 达成提示 | quota 预估 ≥ 目标 | 进度条变绿，显示"达标" |
| 崩溃警告 | fatigue 预计 ≥ 8 | 工人卡红色边框 + 闪烁 |
| 举报警告 | resentment 预计 ≥ 8 | 怨恨连线红色加粗 |

## Forbidden UI
- 不允许只用"让 A 上班/让 B 休息"按钮
- 不允许做公司排班系统界面
- 不允许用纯文字选择列表替代拖卡操作

## Acceptance Rule
- 首屏必须让玩家看到工人卡和岗位槽，可直接拖拽
- 每次拖入必须产生即时预览反馈
- 预览内容必须能追溯到 Required State (quota/fatigue/resentment)
- 不得只靠随机事件文本或按钮选择完成主循环
